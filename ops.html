MYSQL PLSQL

OPERATORS

AND Operator

SELECT * FROM Customers
WHERE Country = 'Germany' AND City = 'Berlin';

Explanation:The above SQL statement selects all fields from "Customers" where country is "Germany" AND city is "Berlin"

OR Operator

SELECT * FROM Customers
WHERE City = 'Berlin' OR City = 'Stuttgart';

Explanation:The above SQL statement selects all fields from "Customers" where city is "Berlin" OR "Stuttgart"

NOT Operator

SELECT * FROM Customers
WHERE NOT Country = 'Germany';

Explanation:The above SQL statement selects all fields from "Customers" where country is NOT "Germany"

IS NULL Operator

SELECT CustomerName, ContactName, Address
FROM Customers
WHERE Address IS NULL;

Explanation:lists all customers with a NULL value in the "Address" field

IS NOT NULL Operator

SELECT CustomerName, ContactName, Address
FROM Customers
WHERE Address IS NOT NULL;

Explanation: lists all customers with a value in the "Address" field

LIKE Operator

SELECT column1, column2, ...
FROM table_name
WHERE columnN LIKE pattern;
EXAMPLE:-
SELECT * FROM Customers
WHERE CustomerName LIKE 'a%';

Explanation:The LIKE operator is used in a WHERE clause to search for a specified pattern in a column.
There are two wildcards often used in conjunction with the LIKE operator:
1.The percent sign (%) represents zero, one, or multiple characters
2.The underscore sign (_) represents one, single character

IN Operator

SELECT * FROM Customers
WHERE Country IN ('Germany', 'France', 'UK');

Explanation:The IN operator allows you to specify multiple values in a WHERE clause.

BETWEEN Operator

SELECT column_name(s)
FROM table_name
WHERE column_name BETWEEN value1 AND value2;

Explanation:The BETWEEN operator selects values within a given range. The values can be numbers, text, or dates.

NOT BETWEEN Operator

SELECT * FROM Products
WHERE Price NOT BETWEEN 10 AND 20;

Explanation:To display the products outside the range of the previous example

UNION Operator

SELECT column_name(s) FROM table1
UNION
SELECT column_name(s) FROM table2;

Explanation:The UNION operator is used to combine the result-set of two or more SELECT statements.
Every SELECT statement within UNION must have the same number of columns
The columns must also have similar data types.
The columns in every SELECT statement must also be in the same order.

EXISTS Operator

SELECT SupplierName
FROM Suppliers
WHERE EXISTS (SELECT ProductName FROM Products WHERE Products.SupplierID = Suppliers.supplierID AND Price < 20);

Explanation:The EXISTS operator is used to test for the existence of any record in a subquery.
The above SQL statement returns TRUE and lists the suppliers with a product price less than 20.

ANY Operator

SELECT ProductName
FROM Products
WHERE ProductID = ANY
  (SELECT ProductID
  FROM OrderDetails
  WHERE Quantity = 10);

Explanation:ANY means that the condition will be true if the operation is true for any of the values in the range.
The above SQL statement lists the ProductName if it finds ANY records in the OrderDetails table has Quantity equal to 10 (this will return TRUE because the Quantity column has some values of 10)

ALL Operator

SELECT ALL ProductName
FROM Products
WHERE TRUE;

Explanation:ALL means that the condition will be true only if the operation is true for all values in the range.
